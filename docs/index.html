Features

🎯 Zod Schemas

Every parameter has a Zod type with .describe() hints. Works with AI structured output, form validation, or plain TypeScript.

🔗 Custom Serializers

Each portal has its own URL serializer. Query params, JSON blobs, path DSLs — we handle the encoding.

📍 Location Codes

Verified neighborhood and city codes. 39 Oslo districts, 25 Manhattan neighborhoods, and more.

🤖 AI-Optimized

Schema descriptions guide AI models to generate valid parameters from natural language queries.

Quick Example

import { finnNoConfig } from "@use_homi/real-estate-portal-schemas";

// Build a rental search URL for Oslo
const url = finnNoConfig.serialize(finnNoConfig.baseUrls.rent!, {
  location: "0.20061",      // Oslo
  price_to: 15000,           // max 15,000 NOK/month
  min_bedrooms: 2,
  facilities: ["1", "4"],    // balcony, elevator
  animals_allowed: "1",      // pets OK
});

// → https://www.finn.no/realestate/lettings/search.html?location=0.20061&price_to=15000&min_bedrooms=2&facilities=1&facilities=4&animals_allowed=1
